## 图片提供网站
------------
#### 简介
这是一个本科生的毕业设计，一个图片提供网站，核心功能是“看图+存图”。

#### 相关信息
作者：chhh
网站名：Tomoz
性质：简单普通的毕业设计
学校：GDUT
完成时间：2018/5/15-2018/5/25

#### 技术方案
###### 数据库
MySQL社区版啊，这有什么好说的。
###### 后台
后台选用了之前用过的国产框架ThinkJS 2.3。
ThinkJS一开始以为用最新3.0版本的，结果发现自己确实有点菜，而且对ES6的一些语法也不熟悉（懒得要死），图省事+省时间就滚去使用2.3版本了。
对这个框架，有很多功能都只是有所了解，大多数时候只是套代码上去用，惭愧。
###### 前端
前端使用了包括 ReactJS、jQuery、zuiJS（UI）、choiJS（edit picture）在内的一些技术。
其中ReactJS由于时间赶、技术菜等原因，使用了babel.js来让浏览器编译，可以说是相当垃圾了。不过，还是有所收获吧。
jQuery用得很少了，由于可能有半年没写代码忘了一些规范什么的，在一开始的代码中胡乱使用了不少jQuery代码，在后期的代码中应该是很少出现jQuery了。
zuiJS这个东西是远光实习的时候用上的，确实挺好用挺好看。他提供了很多好看的样式，节省了我很多时间。
ChoiJS应该是叫这个名字吧，是一个编辑图片的小插件，网上找的，还不错，那部分的代码我都没怎么改。

#### 功能简介
这部分我不太想写，因为太简单太普通了，简单来说，就是用户登录注册模块：登录&注册、主模块：浏览图片&收藏图片、个人空间模块：修改个人信息&调整收藏的图片、编辑模块：编辑图片。
根据后来老师建议的，既然我的这个网站是以图片+文字的方式提升图片价值和实用性，那么可以通过加入站内搜索的功能，深以为然。
另外，老师提出了对用户上传图片的重复性校验的设想，我答辩的时候听完老师讲还是不懂，问同学的话，可能没有问对人，也没人懂，嘤嘤嘤。
还有，老师提出了在我的网站下载的图片，应该载有网站上提供的那些，描述图片的文字，这个可行性贼强。

#### 总结与反思
存在问题：
1、前期评估没做好，一贯以来的问题，在前期准备工作进行时，没有意识到对所选用技术不了解、不熟悉，过度高估了自己的学习和使用能力，导致了在码代码一两天后，才决定更换后台的技术（原本打算使用ExpressJS）和前端react的使用方式（浏览器编译）。
2、码代码的时候，一是刚开始的时候很多代码规范、小技巧都没有用上，二是在写逻辑的时候没有怎么用脑子，基本都是套代码，希望日后看的时候不会骂自己是傻比。
3、毕设初审过了之后就膨胀了，明明还有那么多东西要做，模块没做完、图没画好、逻辑明明可以优化、代码明明可以解耦。
4、明明知道自己的网站还没有给老师过目，老是逃避问题，侥幸心理严重。导致后来快要答辩才去加模块。
